# Variablen und Quoting in Linux

> [!NOTE]
> Weitere Informationen zu Variablen findet ihr in der Manpage der BASH unter der Rubrik  `Parameter Expansion`. Der reguläre Ausdruck für die Suche ist `^ *Parameter Expansion *$`. Für andere Übung müsst ihr die Manpage vielleicht noch ein klein wenig weiter lesen. Schaut euch doch auch mal die Rubriken `Command Substitution` und `Arithmetic Expansion` an.
>
> Hilfe zum Thema Quoting/Escaping findet ihr in der Manpage der `bash` im Abschnitt `QUOTING`.

## Übungen

### 1. Einfache Variable setzen und ausgeben
Setzt eine Variable `name` mit deinem Namen als Wert und gebt sie mit `echo` aus.

### 2. Variable in einer anderen Shell nutzen
Setzt eine Variable `test_var` und überprüft, ob sie in einer komplett neu gestarteten Shell oder in einer Subshell verfügbar ist.

### 3. Mehrzeilige Variable

Speichert den folgenden zweizeiligen Text in einer Variable und gebt ihn aus:

```
Zeile 1: Hallo
Zeile 2: Welt
```

### 4. Umgebungsvariable exportieren
1. Setzt eine Variable `env_var` mit dem Wert `parent` und exportiert sie, sodass sie auch in Subshells verfügbar ist. Ist diese Variable jetzt auch in einer komplett neu gestarteten Shell verfügbar?

2. Mit welchem Kommando könnt ihr überprüfen, ob die Varialbe eine Umgebungsvariable bzw. *Environment Variable* ist?

### 5. Leerzeichen in Variablen
Setzt eine Variable mit einem Wert, der Leerzeichen enthält.

### 6. Zulässige Zeichen in Variablen
Ist es möglich, folgende Namen für Variablen zu erstellen:
```bash
meine variable
meine-variable
1variable
variable?
variable!
var*iable
```

Anders gefragt: Welche Zeichen sind in Variablennamen zulässig bzw. **nicht** zulässig?

### 7. Escpaping/Quoting
1. Setzt eine Variable `somestring`, die folgende Wert haben soll: `Ich bin der Inhalt der Variablen 'somestring'`. Achte darauf, dass auch die einfachen Hochkommata mit ausgegeben werden. Gibt es dafür nur eine Möglichkeit oder mehrere?

2. Gebt folgendes auf der Konsole aus, wobei die Anführungszeichen in der Ausgabe enthalten sollen sein, `$USER` aber substituiert/durch den Usernamen ersetzt werden soll:
```bash
Tux sagt: "Hallo $USER"
```

3. Setzt zunächst folgende Variable:

```bash
name=Ada
```

Führt anschließend diese Befehle aus:

```bash
echo Hallo $name
echo "Hallo $name"
echo 'Hallo $name'
```

Beantwortet danach folgende Fragen:

a) Welche Ausgaben erhaltet ihr?
b) Bei welchem Befehl wird $name durch den Inhalt der Variable ersetzt?
c) Was bewirken einfache Anführungszeichen?
d) Was bewirken doppelte Anführungszeichen?

### 8.1 Escaping/Quoting advanced 1

>[!NOTE]
> Diese und die folgenden zwei Übungen können schon etwas tricky sein. Macht sie also ggf. erst später bzw. macht erstmal mit den anderen Übungen weiter...
 
Warum funktioniert folgendes Kommando nicht? Was passiert da?
```bash
echo 'It's broken'
```

Falls ihr nicht mehr weiterkommt bzw. euren normalen Prompt zurückhaben wollt, probiert einmal die Tastenkombination `STRG+C`. Das ist in der Shell soetwas wie "Abbruch". Genaueres dazu besprechen wir später noch. 

### 8.2 Escaping/Quoting advanced 2

Ihr wollt per SSH auf einem Datenbankserver folgenden SQL-Befehl ausführen:

```
SELECT name FROM users WHERE status = 'active' AND role = 'admin';
```

Der Befehl soll als Argument an das Kommando `mysql -e` übergeben werden. Nun muss jedoch der Befehl der übergeben wird selbst in Anführungszeichen stehen, da er ja Sonderzeichen enthält.

Wie müsste die korrekte Syntax also lauten?

### 8.3 Escaping/Quoting advanced 3

Erstelle eine Variable `MSG`, die exakt folgendes enthält bzw. `echo "$MSG"` exakt diesen String ausgibt:
```bash
It's a "feature", not a bug.
```

### 9. Variablen in Variablen
Erstellt eine Variable namens `mein_home` und weist ihr den Wert der Umgebungsvariable `HOME` zu.

### 10. Verkettung von Variablen

Erstellt eine Variable namens `vorname` und weist ihr den Namen `Ada` zu. Erstellt eine weitere Variable namens `nachname` und weist ihr den Nachnamen `Lovelace` zu. Erstellt eine dritte Variable namens `voller_name` welche den Vornamen und Nachnamen durch ein Leerzeichen getrennt enthalten soll.

Wie viele verschiedene Möglichkeiten findet ihr hier?

### 11. Kombination von Variablen und Text
Erstellt eine Variable `betriebssystem` mit dem Wert `Linux` und eine weitere Variable `wahnsinn`, welcher ihr einen Wert wie `super`, `mega-gut`, `einfach nur klasse` oder `super logisch und eigentlich ganz einfach` zuweist. 

Lasst euch mit `echo` nun folgendes ausgeben: `Ich nutze <betriebssystem>systeme und es ist einfach <wahnsinn>.`. Datei sollen natürlich die beiden Variablen anstelle von `<betriebssytem>` und `<wahnsinn>` genutzt und durch ihren Wert ersetzt werden.

### 12. Variable in Kindprozess prüfen

Erklärt den Unterschied zwischen diesen beiden Aufrufen und versucht voraus zu sagen, was jeweils ausgegeben wird. Werft dazu vielleicht auch einen Blick in die Manpage der `bash` um die Option `-c` zu verstehen:

```bash
# Aufruf 1
FARBE=lila bash -c 'echo $FARBE'

# Aufruf 2
FARBE=lila
bash -c 'echo $FARBE'
```

### 13. Variable mit aktuellem Datum
Setzt eine Variable mit dem Namen `aktuelles_datum` und weist ihr das aktuelle Dateum zu. Nutzt dafür das Kommando `date`. Versucht also, der Variablen die Ausgabe des Kommandos `date` zuzuweisen. Versucht es mal wie folgt: `aktuelles_datum=date`. Das wird nicht funktionieren. Warum? Wie könnt ihr das trotzdem erreichen? 

Tipp: Ihr wollt hier der Variable die Ausgabe eines Kommandos zuweisen, das Kommando soll also durch dessen ausgabe *ersetzt/substituiert* werden...

## Zusatzaufgaben

### 14. Mathematische Operationen mit Variablen
Erstellt zwei Variablen `zahl1` und `zahl2` und weist ihnen beliebige Zahlenwerte zu. Führt nun eine Addition der beiden Variablen durch und speichert das Ergebnis in einer neuen Variable namens `summe`. Gebt den Wert der Variablen `summe` auf der Konsole aus.

### 15. Länge einer Variablen ermitteln
Setzt eine Variable `text` mit einem beliebigen Wert und gib deren Länge aus.

### 16. Teilstring einer Variable extrahieren
Setzt eine Variable `string` mit dem Wert `Linux-Scripting` und extrahiere nur das Wort `Linux`.

### 17. Inhalt einer Variable ändern
Setzt eine Variable `counter` auf `1`, erhöhe sie um `1` und gib den neuen Wert aus.

### 18. Variable mit Standardwert

Schreibe einen Befehl, der die Variable `PORT` ausgibt. Wenn `PORT` nicht gesetzt oder leer ist, soll `8080` als Standardwert verwendet werden - ohne `PORT` dauerhaft zu setzen.
